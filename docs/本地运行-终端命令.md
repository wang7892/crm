# Cordys CRM：终端命令说明

以下均在 **PowerShell** 中示例。请把路径中的 **`E:\CordysCRM-1.6.0`** 换成你本机项目根目录；**`cordys-crm.properties`** 的路径也要换成你的实际绝对路径。

**环境要求：** JDK 21、Node.js（建议 18+）、pnpm、已启动的 MySQL 与 Redis，以及配置正确的 `cordys-crm.properties`。

---

## 1. 安装前端依赖（首次或依赖变更后）

```powershell
Set-Location "E:\CordysCRM-1.6.0\frontend"
pnpm install
```

**作用：** 在 monorepo 根安装依赖，供 `packages/web`、`packages/mobile` 等子包使用。

---

## 2. 启动后端（Spring Boot）

**不要**在仓库根目录使用 `-pl backend/app -am spring-boot:run`（会先对无主类的聚合模块执行插件导致报错）。应对 **`backend/app` 的 POM** 执行：

```powershell
Set-Location "E:\CordysCRM-1.6.0"
.\mvnw.cmd -f backend/app/pom.xml spring-boot:run "-Dspring-boot.run.arguments=--spring.config.additional-location=file:///E:/CordysCRM-1.6.0/cordys-crm.properties"
```

**作用：**

- `.\mvnw.cmd`：使用项目自带 Maven。
- `-f backend/app/pom.xml`：只对可执行应用模块执行 `spring-boot:run`，避免根 POM / `backend` 聚合 POM 触发「找不到 main class」。
- `--spring.config.additional-location=...`：加载数据库、Redis 等外部配置；**`file:///E:/...`** 须为真实存在的文件 URL（三斜杠 + 盘符路径）。

默认服务端口一般为 **8081**（见 `commons.properties`）。

---

## 3. 启动 Web 前端（开发）

新开一个终端：

```powershell
Set-Location "E:\CordysCRM-1.6.0\frontend\packages\web"
npm run dev
```

**作用：** 启动 Vite 开发服务（常见为 `http://localhost:5173`）。通过 `packages/web/.env.development` 中的 `VITE_DEV_DOMAIN` 将 `/front` 等代理到后端（默认 `http://127.0.0.1:8081`），**需先启动后端**。

等价（在 `frontend` 目录）：

```powershell
Set-Location "E:\CordysCRM-1.6.0\frontend"
pnpm --filter @cordys/web dev
```

---

## 4. 启动 Mobile 前端（开发）

再开一个终端：

```powershell
Set-Location "E:\CordysCRM-1.6.0\frontend\packages\mobile"
npm run dev
```

**作用：** 启动移动端 Vite（配置中为 **3000** 端口）。同样依赖后端与 `VITE_DEV_DOMAIN`；模拟登录见官方文档（从 Web 端复制 `localStorage` 中的 `sessionId`、`csrfToken`）。

等价：

```powershell
Set-Location "E:\CordysCRM-1.6.0\frontend"
pnpm --filter @cordys/mobile dev
```

---

## 5. 仅构建前端（不启动）

```powershell
Set-Location "E:\CordysCRM-1.6.0\frontend"
pnpm install
pnpm run build
```

**作用：** 递归构建各前端子包，生成各包下的 `dist`。

---

## 6. 完整 Maven 打包（前端 + 后端，生成 jar）

```powershell
Set-Location "E:\CordysCRM-1.6.0"
.\mvnw.cmd clean package -DskipTests
```

**作用：** `clean` 清理产物；`package` 构建根工程（含前端插件执行 `pnpm install` / `pnpm run build`）与后端，得到 **`backend\app\target\app-main.jar`**。`-DskipTests` 跳过测试；若要跑测试，去掉该参数。

---

## 7. 运行已打包的 jar

```powershell
java -jar "E:\CordysCRM-1.6.0\backend\app\target\app-main.jar" --spring.config.additional-location=file:///E:/CordysCRM-1.6.0/cordys-crm.properties
```

**作用：** 使用 JDK 21 运行可执行包，并加载外部配置（路径按需修改）。

---

## 建议启动顺序

1. MySQL、Redis 就绪，`cordys-crm.properties` 已填写正确。  
2. 执行 **第 2 节** 启动后端。  
3. 再执行 **第 3 节** / **第 4 节** 启动前端。

---

## 补充说明

- **Windows** 使用 **`.\mvnw.cmd`**，不要用 `./mvnw`（除非在 Git Bash 等环境）。  
- **`additional-location` 的 `file:///` 路径** 必须与文件真实位置一致，否则 Spring 读不到配置。  
- 若需在本机覆盖日志目录，可在 `cordys-crm.properties` 中设置 `logging.file.path` 为 Windows 可写路径。
